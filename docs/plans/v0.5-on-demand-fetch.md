# v0.5 On-Demand Fetch

`shopify-rextant` can recover a missing official Shopify documentation page when the caller already knows the exact Shopify docs URL or canonical docs path.

## Enablement

On-demand network access is disabled by default. Enable it in the configured home directory:

```toml
[index]
enable_on_demand_fetch = true
```

For the default home, the file is `~/.shopify-rextant/config.toml`.

## Accepted Scope

Only these official Shopify developer documentation scopes are accepted:

- `https://shopify.dev/docs/**`
- `https://shopify.dev/changelog/**`

Other hosts, schemes, and Shopify paths are rejected before any network request.

## MCP Usage

```json
{
  "name": "shopify_fetch",
  "arguments": {
    "url": "https://shopify.dev/docs/apps/build/access-scopes"
  }
}
```

If enabled, the server fetches the raw `.md` or `.txt` source candidate, saves it under `data/raw/`, upserts the SQLite `docs` row with `source = "on_demand"` for newly recovered docs, and updates the Tantivy index for that canonical path.

If disabled, MCP returns JSON-RPC error `-32007` with the candidate URL and `enable_on_demand_fetch=false`.

If outside scope, MCP returns JSON-RPC error `-32008`.

## CLI Usage

```bash
shopify-rextant refresh --url https://shopify.dev/docs/apps/build/access-scopes
```

Existing positional refresh behavior remains available:

```bash
shopify-rextant refresh /docs/apps/build/access-scopes
shopify-rextant refresh
```

To retry failed coverage rows:

```bash
shopify-rextant coverage repair
```

Repair uses the same policy gate. Disabled config skips allowed rows without fetching; disallowed URLs are skipped by policy.
